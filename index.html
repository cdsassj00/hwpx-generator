Claude Skill → npx → 브라우저 CDN으로 이어지는 여정

1. Skill이 뭐예요?

Skill은 Claude(AI)가 특정 작업을 잘 하도록 모아놓은 "전문 지식 꾸러미"예요. 한글 보고서를 잘 만드는 방법, 엑셀을 다루는 방법, PDF를 만드는 방법 — 각각이 하나의 스킬입니다.

예를 들어 "HWPX 파일을 정부 공문 스타일로 만들어줘" 같은 요청은 일반 AI가 잘 못합니다. 그런데 hwpx-gov-doc이라는 스킬을 Claude에게 "장착"해주면, 그 안에 들어있는 템플릿·규칙·코드 덕분에 전문가처럼 만들어줘요.

비유 — 요리사에게 "백종원 레시피북"을 건네주는 것과 같아요. 요리사는 원래도 요리를 하지만, 그 책을 보면 특정 요리를 훨씬 잘 만듭니다.

2. Skill은 어디서 받나요?

  1. Anthropic 공식 저장소 — GitHub의 Anthropic skills 레포에 여러 스킬이 공개돼 있어요
  2. Claude Code Plugin Marketplace — Claude Code에서 플러그인 형태로 설치
  3. 직접 만들기 — SKILL.md 파일과 도구를 폴더로 묶으면 스킬이 됨

3. 이 사이트는 어떻게 만들어졌어요?

원래 hwpx-gov-doc 스킬은 Node.js(개발자용 프로그램)에서 돌아가는 도구였어요. 그래서 평범한 공무원이 쓰기 어려웠죠. "웹 브라우저에서 바로 쓰게 해보자"가 이 프로젝트의 목적입니다.

[1단계] 스킬 받기 GitHub에서 hwpx-gov-doc 스킬 다운로드 ↓ [2단계] Node.js 코드 → 브라우저 코드로 변환 npx로 돌리던 걸 → 순수 JavaScript로 포팅 ↓ [3단계] ZIP 처리 라이브러리 붙이기 JSZip CDN으로 브라우저에서 HWPX(=ZIP) 조작 ↓ [4단계] Claude AI 연결 "문서 주제"를 자연어로 입력 → AI가 구조화 ↓ [5단계] 최종: 버튼 하나로 HWPX 다운로드 🎉

4. 이 사이트에서 할 수 있는 것

  • AI 보고서 — "디지털 전환 보고서 만들어줘" 같은 자연어 요청 → Claude가 섹션 구조 짜고 → HWPX로 자동 변환
  • 내 양식 채우기 — 기존 빈 HWPX 양식(착수계·공문 등) 업로드 → 핵심 내용만 쓰면 → AI가 빈칸 자동 채워서 다운로드
핵심 철학 — 공무원이 이미 쓰고 있는 한글(HWPX) 파일을 그대로 지원해서, AI를 업무에 녹여내는 것. 새로운 것을 배우지 않아도 됩니다.

1. Skill → 브라우저 포팅 과정

원본 hwpx-gov-doc 스킬은 Node.js 모듈이었습니다. python-docx스러운 API로 HWPX ZIP 안의 XML을 조립하는 구조.

브라우저로 옮기려면 두 가지만 바꾸면 됩니다:

  1. fs → Blob + URL.createObjectURL
  2. JSZip (Node) → JSZip (CDN UMD) — 동일 API라 거의 그대로
// 원본 (Node.js)
const fs = require('fs');
const JSZip = require('jszip');
const zip = new JSZip();
zip.file('Contents/section0.xml', xml);
fs.writeFileSync('out.hwpx', await zip.generateAsync({type:'nodebuffer'}));

// 브라우저 이식
<script src="https://cdn.jsdelivr.net/npm/jszip"></script>
const zip = new JSZip();
zip.file('Contents/section0.xml', xml);
const blob = await zip.generateAsync({
  type: 'blob',
  mimeType: 'application/hwp+zip',
  compression: 'DEFLATE'
});
URL.createObjectURL(blob); // 다운로드 링크 생성

2. Streaming SSE via Edge Function

Netlify 무료 플랜의 sync function은 10초 타임아웃이라 Sonnet/Opus가 자주 504를 뱉었습니다. 해결은 Edge Function + SSE pass-through:

// netlify/edge-functions/chat.ts
export default async (req: Request) => {
  const body = await req.json();
  const apiKey = Netlify.env.get('CLAUDE_API_KEY');
  const r = await fetch('https://api.anthropic.com/v1/messages', {
    method: 'POST',
    headers: { 'x-api-key': apiKey!, 'anthropic-version': '2023-06-01',
               'Content-Type': 'application/json' },
    body: JSON.stringify({ ...body, stream: true }),
  });
  return new Response(r.body, {
    headers: {
      'Content-Type': 'text/event-stream',
      'Cache-Control': 'no-cache, no-transform',
      'X-Accel-Buffering': 'no',
    },
  });
};

브라우저는 ReadableStream을 직접 파싱:

const reader = resp.body.getReader();
const decoder = new TextDecoder();
let buf = '';
while (true) {
  const { done, value } = await reader.read(); if (done) break;
  buf += decoder.decode(value, { stream: true });
  const lines = buf.split('\n'); buf = lines.pop() || '';
  for (const line of lines) {
    if (!line.startsWith('data: ')) continue;
    const evt = JSON.parse(line.slice(6));
    if (evt.type === 'content_block_delta' && evt.delta?.type === 'text_delta') {
      aiText += evt.delta.text;
    }
  }
}

3. HWPX 양식 자동 채우기 (2025년 추가)

XML의 <hp:t> 텍스트 노드를 정규식으로 추출하고, 각 노드를 4가지로 분류:

  • label — 한글 레이블 ("용 역 명", "상 호") → 불변
  • blank — 순수 whitespace → 값으로 통째 교체
  • mixed — 레이블+공백 혼합 (": 금 원(금 원)") → 공백 영역에만 값 삽입, 구조 보존
  • body / tab — 본문·hp:tab 제어 → 불변

AI에는 fillable 후보 + 인근 label context만 전달. 응답 후 서버 측 검증으로 label 노드를 건드린 fill은 거부하고, 같은 ctx의 중복 필드는 자동 복제 (e.g. 6개 서류 반복되는 상호·대표자).
